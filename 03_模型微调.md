# 03_模型微调

## 什么是模型微调？

**一句话**：在预训练模型基础上，用特定数据进一步训练，让它更擅长特定任务

**类比**：
```
预训练 = 大学毕业（通用知识）
微调 = 职业培训（专业技能）

通用GPT → 医疗GPT（医疗数据微调）
通用GPT → 代码GPT（代码数据微调）
```

---

## 微调方法对比

| 方法 | 参数量 | 显存需求 | 效果 | 适用场景 |
|------|--------|----------|------|----------|
| 全参数微调 | 100% | 极高 | 最好 | 资源充足 |
| LoRA | 0.1%-1% | 低 | 好 | 通用场景 |
| QLoRA | 0.1%-1% | 更低 | 好 | 显存有限 |
| Adapter | 1%-5% | 中 | 中等 | 多任务 |

## LoRA详解

### 什么是LoRA？

**全称**：Low-Rank Adaptation（低秩适配）

**核心思想**：大模型的参数更新是"低秩"的，可以用小矩阵近似

```
原始权重：W (4096×4096) = 1600万参数
LoRA：W + A×B
A (4096×16) + B (16×4096) = 13万参数（仅0.8%）
```

**类比**：
```
全参数微调：重新装修整个房子
LoRA：只换窗帘和沙发套，效果差不多但省很多钱
```

### 什么是"低秩"？

**一句话**：矩阵看起来很大，但实际信息量很小

```
一个 1000×1000 的矩阵（100万个数字）
如果秩只有10，只需要20个数字就能表示

低秩分解：1000×10 + 10×1000 = 20,000 参数
压缩了50倍！
```

### LoRA代码

```python
from peft import LoraConfig, get_peft_model

# LoRA配置
lora_config = LoraConfig(
    r=16,                    # 秩
    lora_alpha=32,           # 缩放因子
    target_modules=["q_proj", "v_proj"],  # 应用到哪些层
    lora_dropout=0.1
)

# 应用LoRA
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# trainable params: 4M || all params: 7B || trainable%: 0.06%
```

## QLoRA

### 什么是QLoRA？

**一句话**：4-bit量化 + LoRA，显存需求更低

**效果**：
```
LLaMA-7B：
- 全精度：14GB显存
- QLoRA：5GB显存（可训练）
```

### QLoRA代码

```python
from transformers import BitsAndBytesConfig
from peft import LoraConfig, get_peft_model

# 量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.float16
)

# 加载量化模型
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    quantization_config=bnb_config
)

# 应用LoRA
model = get_peft_model(model, lora_config)
```

## 训练流程

### 完整流程

```
1. 准备数据
   ↓
2. 加载模型和Tokenizer
   ↓
3. 配置LoRA/QLoRA
   ↓
4. 设置训练参数
   ↓
5. 训练
   ↓
6. 保存模型
   ↓
7. 测试和部署
```

### 数据格式

```json
{
  "instruction": "将以下英文翻译成中文",
  "input": "Hello, how are you?",
  "output": "你好，你好吗？"
}
```

### 训练代码

```python
from transformers import TrainingArguments, Trainer

# 训练参数
training_args = TrainingArguments(
    output_dir="./results",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    learning_rate=2e-4,
    fp16=True
)

# 训练
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=dataset
)
trainer.train()

# 保存
model.save_pretrained("./lora_weights")
```

## RLHF与DPO

### 什么是RLHF？

**全称**：Reinforcement Learning from Human Feedback

**核心思想**：让人类的偏好指导模型学习

```
ChatGPT = 预训练 + 指令微调 + RLHF
```

### RLHF流程

```
步骤1：指令微调（SFT）
  模型学习遵循指令
  ↓
步骤2：训练奖励模型（RM）
  人类标注：回答A > 回答B
  ↓
步骤3：强化学习（PPO）
  用奖励模型指导生成
```

### 什么是DPO？

**全称**：Direct Preference Optimization

**核心思想**：跳过奖励模型，直接学习偏好

```
RLHF：偏好数据 → 奖励模型 → 强化学习
DPO：偏好数据 → 直接优化

DPO更简单，效果相当
```

### DPO代码

```python
from trl import DPOTrainer, DPOConfig

# 偏好数据格式
# {"prompt": "问题", "chosen": "好的回答", "rejected": "差的回答"}

trainer = DPOTrainer(
    model=model,
    ref_model=ref_model,
    args=DPOConfig(learning_rate=5e-5),
    train_dataset=dataset
)
trainer.train()
```

## 实战：训练客服助手

### 数据准备

```json
[
  {
    "instruction": "回答用户关于退货的问题",
    "input": "我买的东西不满意，可以退货吗？",
    "output": "您好！我们支持7天无理由退货。请在订单详情页申请退货。"
  }
]
```

### 训练代码

```python
# 1. 加载模型（4-bit量化）
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2-7B",
    quantization_config=BitsAndBytesConfig(load_in_4bit=True)
)

# 2. 配置LoRA
model = get_peft_model(model, LoraConfig(r=16))

# 3. 训练
trainer = Trainer(model=model, args=training_args, train_dataset=dataset)
trainer.train()

# 4. 测试
prompt = "退货问题：我买的东西不满意"
inputs = tokenizer(prompt, return_tensors="pt")
outputs = model.generate(**inputs, max_length=200)
print(tokenizer.decode(outputs[0]))
```
