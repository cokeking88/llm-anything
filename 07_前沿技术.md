## 1. 长上下文技术

### 什么是长上下文？

**一句话**：让模型能处理更长的文本

**对比**：
```
GPT-3.5：4K token（约3000字）
GPT-4：128K token（约10万字）
Gemini 1.5：1M token（约75万字）
```

### 核心技术

**RoPE位置编码改进**：
```
传统RoPE：固定位置编码
NTK-aware：动态调整
YaRN：进一步优化
```

**Ring Attention**：
```
分布式计算，每个GPU处理一部分
突破单卡显存限制
```

**StreamingLLM**：
```
保留开头和最近的token
中间的token可以丢弃
支持无限长输入
```

## 2. MoE架构

### 什么是MoE？

**全称**：Mixture of Experts（混合专家）

**一句话**：每次只激活一部分参数

**类比**：
```
传统模型：每次用所有参数
MoE模型：根据输入选择最相关的"专家"

像医院：不同病找不同科室的医生
```

### 代表模型

| 模型 | 总参数 | 激活参数 | 效果 |
|------|--------|----------|------|
| Mixtral 8x7B | 46.7B | 12.9B | 接近Llama-2 70B |
| DeepSeek-V2 | 236B | 21B | 性价比极高 |

### 优势

```
总参数多 → 能力强
激活参数少 → 速度快
训练成本低 → 更经济
```

## 3. 小模型优化

### 蒸馏（Distillation）

**一句话**：用大模型教小模型

```
教师模型（大）→ 学生模型（小）
大模型输出 → 小模型学习模仿
```

**代表**：
- Gemma 2B（Google）
- Phi-3 Mini（Microsoft）
- Qwen2-0.5B（阿里）

### 量化（Quantization）

**INT4量化**：
```
7B模型：14GB → 4GB
速度提升2-3倍
精度损失很小
```

**GPTQ/AWQ**：
```
训练后量化
支持4-bit/3-bit/2-bit
```

### 剪枝（Pruning）

**结构化剪枝**：
```
移除不重要的注意力头
移除不重要的层
模型变小变快
```

## 4. 推理优化

### 投机解码（Speculative Decoding）

**一句话**：用小模型快速猜测，大模型验证

```
小模型：快速生成多个token
大模型：一次性验证所有token
如果对了，直接用；错了，重新生成

速度提升2-3倍
```

### vLLM

**核心创新**：PagedAttention

```
传统：显存浪费严重
PagedAttention：像操作系统管理内存一样管理KV Cache

吞吐量提升24倍
```

### SGLang

**核心创新**：快速推理框架

```
支持RadixAttention（前缀缓存）
支持自动批处理
支持结构化输出

速度比vLLM更快
```

## 5. 对齐技术

### Constitutional AI

**核心思想**：让AI自己监督自己

```
1. AI生成回答
2. AI根据原则批评自己
3. AI根据批评改进
4. 重复直到满意
```

### KTO（Kahneman-Tversky Optimization）

**比DPO更简单**：
```
DPO：需要成对数据（好的+坏的）
KTO：只需要标注好坏

数据准备更简单
```

### ORPO（Odds Ratio Preference Optimization）

**不需要参考模型**：
```
DPO：需要参考模型
ORPO：直接优化

训练更简单，效果相当
```

## 6. 多Agent前沿

### AutoGPT / AgentGPT

**自主执行复杂任务**：
```
目标：写一份市场分析报告
Agent自动：
1. 搜索信息
2. 分析数据
3. 撰写报告
4. 检查修改
```

### CrewAI

**多Agent协作**：
```
团队组成：
- 研究员Agent
- 分析师Agent
- 写手Agent
- 编辑Agent

分工合作完成任务
```

### MetaGPT

**软件开发Agent**：
```
输入：产品需求
输出：完整代码

Agent团队：
- 产品经理
- 架构师
- 程序员
- 测试员
```
