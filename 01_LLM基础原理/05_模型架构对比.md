# 模型架构对比

## 模型架构分类

### 主要分类
```
1. Encoder-only（编码器-only）
2. Decoder-only（解码器-only）
3. Encoder-Decoder（编码器-解码器）
```

## Encoder-only（BERT类）

### 架构特点
```
结构：只有编码器
注意力：双向注意力
输入：整个序列同时处理
输出：每个位置的表示
```

### 工作原理
```
输入序列 → 编码器 → 上下文表示
每个token都可以看到所有其他token
```

### 代表模型
```
BERT：Google，2018年
RoBERTa：Facebook，2019年
ALBERT：Google，2019年
DeBERTa：Microsoft，2020年
```

### 适用任务
```
1. 文本分类
   - 情感分析
   - 垃圾邮件检测
   - 主题分类

2. 命名实体识别（NER）
   - 识别人名、地名、组织名
   - 信息抽取

3. 语义相似度
   - 问答匹配
   - 文档相似度

4. 关系抽取
   - 实体关系识别
   - 知识图谱构建
```

### 优缺点
```
优点：
  - 双向理解，上下文信息丰富
  - 适合理解类任务
  - 训练相对稳定

缺点：
  - 不适合生成任务
  - 需要特殊标记（[CLS]、[SEP]）
  - 推理时需要整个输入
```

### 代码示例
```python
from transformers import BertTokenizer, BertModel

# 加载模型
tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
model = BertModel.from_pretrained('bert-base-uncased')

# 编码输入
text = "Hello, how are you?"
inputs = tokenizer(text, return_tensors="pt")

# 获取表示
outputs = model(**inputs)
last_hidden_states = outputs.last_hidden_state

# [CLS] token的表示用于分类
cls_representation = last_hidden_states[:, 0, :]
```

## Decoder-only（GPT类）

### 架构特点
```
结构：只有解码器
注意力：因果注意力（掩码）
输入：自回归生成
输出：下一个token的概率
```

### 工作原理
```
输入序列 → 解码器 → 下一个token
每个token只能看到之前的token
```

### 代表模型
```
GPT系列：OpenAI
LLaMA系列：Meta
Qwen系列：阿里
DeepSeek：深度求索
Falcon：TII
```

### 适用任务
```
1. 文本生成
   - 故事创作
   - 文章写作
   - 诗歌生成

2. 对话系统
   - 聊天机器人
   - 客服助手
   - 个人助理

3. 代码生成
   - 代码补全
   - 代码生成
   - 代码解释

4. 推理任务
   - 数学计算
   - 逻辑推理
   - 问题解决
```

### 优缺点
```
优点：
  - 天然适合生成任务
  - 自回归生成质量高
  - 可以处理任意长度序列

缺点：
  - 无法看到未来信息
  - 推理速度较慢（需要逐步生成）
  - 理解任务可能不如双向模型
```

### 代码示例
```python
from transformers import GPT2Tokenizer, GPT2LMHeadModel

# 加载模型
tokenizer = GPT2Tokenizer.from_pretrained('gpt2')
model = GPT2LMHeadModel.from_pretrained('gpt2')

# 编码输入
text = "The weather today is"
inputs = tokenizer(text, return_tensors="pt")

# 生成文本
outputs = model.generate(
    **inputs,
    max_length=50,
    temperature=0.7,
    do_sample=True
)

# 解码输出
generated_text = tokenizer.decode(outputs[0], skip_special_tokens=True)
```

## Encoder-Decoder（T5类）

### 架构特点
```
结构：编码器 + 解码器
注意力：编码器双向，解码器因果
输入：编码器处理输入序列
输出：解码器生成输出序列
```

### 工作原理
```
输入序列 → 编码器 → 上下文表示 → 解码器 → 输出序列
编码器理解输入，解码器生成输出
```

### 代表模型
```
T5：Google，2020年
BART：Facebook，2019年
mT5：Google，2020年
Pegasus：Google，2020年
```

### 适用任务
```
1. 机器翻译
   - 英语到中文
   - 多语言翻译

2. 文本摘要
   - 文章摘要
   - 对话摘要

3. 问答系统
   - 阅读理解
   - 开放域问答

4. 文本改写
   - 语法纠错
   - 风格转换
```

### 优缺点
```
优点：
  - 输入和输出可以不同长度
  - 适合序列到序列任务
  - 理解和生成能力平衡

缺点：
  - 参数量较大
  - 训练相对复杂
  - 推理需要两步
```

### 代码示例
```python
from transformers import T5Tokenizer, T5ForConditionalGeneration

# 加载模型
tokenizer = T5Tokenizer.from_pretrained('t5-small')
model = T5ForConditionalGeneration.from_pretrained('t5-small')

# 编码输入
input_text = "translate English to German: How are you?"
input_ids = tokenizer(input_text, return_tensors="pt").input_ids

# 生成输出
outputs = model.generate(input_ids, max_length=50)

# 解码输出
generated_text = tokenizer.decode(outputs[0], skip_special_tokens=True)
# "Wie geht es Ihnen?"
```

## 架构对比总结

### 技术对比表
| 特性 | Encoder-only | Decoder-only | Encoder-Decoder |
|------|--------------|--------------|-----------------|
| 注意力类型 | 双向 | 因果（单向） | 编码器双向，解码器因果 |
| 代表模型 | BERT、RoBERTa | GPT、LLaMA | T5、BART |
| 主要任务 | 理解 | 生成 | 转换 |
| 输入处理 | 全局 | 自回归 | 编码器全局，解码器自回归 |
| 输出形式 | 每个位置的表示 | 下一个token | 完整序列 |

### 性能对比
```
理解任务：
  Encoder-only > Encoder-Decoder > Decoder-only
  
生成任务：
  Decoder-only > Encoder-Decoder > Encoder-only
  
转换任务：
  Encoder-Decoder > Decoder-only > Encoder-only
```

### 资源需求对比
```
参数量：
  Encoder-only: 中等
  Decoder-only: 可大可小
  Encoder-Decoder: 较大

训练成本：
  Encoder-only: 中等
  Decoder-only: 中等
  Encoder-Decoder: 较高

推理速度：
  Encoder-only: 快
  Decoder-only: 慢（需要生成）
  Encoder-Decoder: 中等
```

## 选择建议

### 选择Encoder-only的情况
```
场景：文本理解任务
任务：分类、NER、相似度
数据：有标注数据
资源：中等计算资源
```

### 选择Decoder-only的情况
```
场景：文本生成任务
任务：对话、创作、代码生成
数据：大规模无标注数据
资源：充足计算资源
```

### 选择Encoder-Decoder的情况
```
场景：序列转换任务
任务：翻译、摘要、改写
数据：有标注的成对数据
资源：充足计算资源
```

## 发展趋势

### 1. Decoder-only主导
```
趋势：GPT类模型成为主流
原因：生成能力重要，扩展性好
```

### 2. 多模态扩展
```
趋势：支持文本、图像、音频
模型：GPT-4V、Gemini、LLaVA
```

### 3. 高效架构
```
趋势：减少参数，提高效率
方法：稀疏化、量化、蒸馏
```

### 4. 专用架构
```
趋势：为特定领域优化
示例：代码模型、多语言模型
```

## 实际应用选择

### 企业应用
```
推荐：Decoder-only（如GPT-4）
理由：能力强，支持多种任务
替代：开源模型（如LLaMA）
```

### 研究应用
```
推荐：根据任务选择
理解任务：Encoder-only
生成任务：Decoder-only
转换任务：Encoder-Decoder
```

### 边缘设备
```
推荐：小型Decoder-only
示例：Phi-2、Gemma-2B
考虑：参数量、推理速度
```
