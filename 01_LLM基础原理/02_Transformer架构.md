# Transformer架构详解

## 什么是Transformer？

### 一句话定义
**一句话**：基于"注意力机制"的架构，能并行处理序列

### 类比理解
```
RNN：像读书，一个字一个字读（串行）
Transformer：像拍照，一眼看到所有字（并行）

RNN：记忆力有限，前面的容易忘
Transformer：每个字都能直接看到所有字
```

## Transformer的诞生背景

### 传统RNN的局限
```
问题1：串行计算，无法并行化
问题2：长距离依赖问题（梯度消失/爆炸）
问题3：训练速度慢
```

### Transformer的突破
```
突破1：完全基于注意力机制，支持并行计算
突破2：直接建立长距离依赖关系
突破3：训练速度快，效果更好
```

## 核心：注意力机制

### 什么是注意力机制？

**直观理解**：注意力就是"关注重点"

```
句子："小明在北京大学学习人工智能"

当处理"学习"时，注意力分配：
- 小明：0.3（主语）
- 北京大学：0.2（地点）
- 学习：0.4（当前词）
- 人工智能：0.1（宾语）
```

### 注意力计算公式
```
Attention(Q, K, V) = softmax(Q × Kᵀ / √dₖ) × V

Q：Query（我在找什么）
K：Key（我能提供什么）
V：Value（我的内容）
dₖ：Key的维度（用于缩放）
```

### 注意力计算步骤

#### 步骤1：线性变换
```
输入X → 线性变换得到Q、K、V
Q = X × W_Q
K = X × W_K
V = X × W_V
```

#### 步骤2：计算注意力分数
```
注意力分数 = Q × Kᵀ
每个词对其他词的关注程度
```

#### 步骤3：缩放和归一化
```
缩放：注意力分数 / √dₖ
Softmax：归一化为概率分布
```

#### 步骤4：加权求和
```
输出 = 注意力权重 × V
```

## 自注意力机制（Self-Attention）

### 什么是自注意力？
```
定义：序列中的每个元素都与序列中的所有元素计算注意力
特点：没有顺序信息，需要位置编码
```

### 自注意力图解
```
输入：I love you

每个词都和其他词计算相关性：
        I    love   you
I      1.0    0.3   0.2
love   0.3    1.0   0.8
you    0.2    0.8   1.0

"love"最关注"you"（0.8）
```

### 多头注意力（Multi-Head Attention）
```
概念：并行运行多个注意力机制，每个关注不同的方面
优点：捕捉不同类型的依赖关系
示例：
  头1：关注语法关系
  头2：关注语义关系
  头3：关注位置关系
```

## 位置编码（Positional Encoding）

### 为什么需要位置编码？
```
问题：注意力机制本身没有位置信息
示例：
  "猫吃鱼" 和 "鱼吃猫"
  没有位置编码，两个句子表示相同
```

### 位置编码方法

#### 正弦位置编码
```
PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

优点：可以处理任意长度的序列
```

#### 可学习位置编码
```
方法：为每个位置学习一个向量
优点：简单，效果不错
缺点：最大长度固定
```

## 前馈神经网络（Feed Forward）

### 作用
```
对每个位置独立进行非线性变换
增加模型的表达能力
```

### 结构
```
FFN(x) = max(0, xW₁ + b₁)W₂ + b₂
激活函数：ReLU或GELU
```

## 残差连接和层归一化

### 残差连接（Residual Connection）
```
作用：缓解梯度消失，加速训练
公式：output = LayerNorm(x + Sublayer(x))
```

### 层归一化（Layer Normalization）
```
作用：稳定训练，加速收敛
方法：对每个样本的特征维度进行归一化
```

## Encoder和Decoder的区别

### Encoder（编码器）
```
功能：理解输入序列
特点：双向注意力，可以看到所有位置
应用：BERT、文本理解任务
```

### Decoder（解码器）
```
功能：生成输出序列
特点：掩码注意力，只能看到之前的位置
应用：GPT、文本生成任务
```

### 掩码注意力（Masked Attention）
```
作用：防止解码器看到未来的位置
实现：将未来位置的注意力分数设为-∞
效果：确保自回归生成
```

## Transformer的优势

### 1. 并行计算
```
优势：所有位置同时计算，训练速度快
对比：RNN需要串行计算，速度慢
```

### 2. 长距离依赖
```
优势：任意两个位置直接连接
对比：RNN需要逐步传递，信息丢失
```

### 3. 可扩展性
```
优势：可以扩展到数十亿参数
应用：GPT-3、GPT-4、LLaMA等
```

## 实际应用

### 不同架构的应用
```
Encoder-only（BERT类）：
  - 文本分类
  - 命名实体识别
  - 语义相似度

Decoder-only（GPT类）：
  - 文本生成
  - 对话系统
  - 代码生成

Encoder-Decoder（T5类）：
  - 机器翻译
  - 文本摘要
  - 问答系统
```
