# 什么是大语言模型（LLM）

## 一句话定义
**一句话**：通过阅读海量文本学会"说话"的AI

## 类比理解
```
传统NLP：教小孩做数学题（规则明确）
LLM：教小孩读万卷书后写文章（从数据中学习）

ChatGPT = 读了互联网上几乎所有文字后学会的"超级学生"
```

## LLM的发展历程

### 早期AI（规则系统）
```
规则：如果包含"天气"，则返回天气预报
问题：需要手动编写所有规则
局限：无法处理复杂、模糊的语言
```

### 统计机器学习
```
方法：从数据中学习统计规律
代表：n-gram、SVM
局限：无法理解深层语义
```

### 深度学习时代
```
突破：神经网络学习复杂模式
代表：RNN、LSTM、Transformer
特点：端到端学习，自动特征提取
```

### 大语言模型时代
```
规模：数十亿到数千亿参数
数据：万亿级token训练数据
能力：涌现出理解、推理、创造能力
```

## LLM的能力边界

### LLM擅长的能力
1. **语言理解**：理解文本的语义和意图
2. **文本生成**：生成连贯、有逻辑的文本
3. **知识问答**：基于训练知识回答问题
4. **代码生成**：理解和生成各种编程语言
5. **推理能力**：简单的逻辑推理和数学计算

### LLM的局限性
1. **幻觉问题**：可能生成看似合理但错误的内容
2. **知识截止**：训练数据有截止日期
3. **计算限制**：无法进行复杂计算
4. **实时性**：无法获取实时信息
5. **推理深度**：复杂的多步推理仍有困难

## LLM的工作原理

### 核心思想：预测下一个词
LLM的本质是一个**概率模型**，它学习的是：

```
给定前文W = w₁, w₂, ..., wₙ，预测下一个词wₙ₊₁的概率
P(wₙ₊₁ | w₁, w₂, ..., wₙ)
```

### 生成过程
```
输入：今天天气
模型预测：真 (概率0.8)
输入：今天天气真
模型预测：好 (概率0.9)
输入：今天天气真好
模型预测：， (概率0.7)
...
逐步生成完整句子
```

## LLM的分类

### 按参数规模分类
| 类别 | 参数量 | 代表模型 | 特点 |
|------|--------|----------|------|
| 小模型 | <1B | Phi-2, Gemma-2B | 轻量、快速、适合边缘设备 |
| 中模型 | 1B-10B | LLaMA-7B, Qwen-7B | 平衡性能和资源 |
| 大模型 | 10B-100B | LLaMA-70B, Qwen-72B | 强大、需要多GPU |
| 超大模型 | >100B | GPT-4, Claude 3.5 | 最强能力、闭源 |

### 按架构分类
1. **Decoder-only**：GPT系列、LLaMA系列
   - 擅长文本生成
   - 自回归生成
   
2. **Encoder-only**：BERT系列
   - 擅长文本理解
   - 双向编码
   
3. **Encoder-Decoder**：T5、BART
   - 擅长序列转换
   - 编码-解码结构

## 关键术语解释

### Token（词元）
```
概念：LLM处理的基本单位
特点：可能是词、词的一部分、或单个字符
示例：
  "我爱人工智能" → ["我", "爱", "人工", "智能"]
  "I love AI" → ["I", "love", " ", "AI"]
```

### 上下文窗口
```
定义：模型能处理的最大token数量
重要性：决定了模型能"记住"多少信息
示例：
  GPT-3.5：4K token（约3000字）
  GPT-4：128K token（约10万字）
  Gemini 1.5：1M token（约75万字）
```

### 温度（Temperature）
```
作用：控制生成的随机性
范围：0-2（通常0.7-1.0）
效果：
  温度低（0.1-0.3）：确定性强，重复性高
  温度中（0.7-1.0）：平衡创造性和准确性
  温度高（1.5-2.0）：创造性强，可能不连贯
```
