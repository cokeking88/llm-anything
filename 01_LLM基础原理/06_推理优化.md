# 推理优化技术

## 为什么需要推理优化？

### 挑战
```
1. 模型太大：参数数十亿到数千亿
2. 显存不足：单卡无法容纳大模型
3. 推理速度慢：生成延迟高
4. 成本高昂：部署成本大
```

### 优化目标
```
1. 减少显存占用
2. 加快推理速度
3. 降低部署成本
4. 保持模型效果
```

## 量化（Quantization）

### 什么是量化？
**核心思想**：用更少的位数表示参数

### 量化原理
```
原始：FP32（32位浮点数）
量化：FP16/BF16（16位）→ INT8（8位）→ INT4（4位）

每个参数：
  FP32：4字节
  FP16：2字节
  INT8：1字节
  INT4：0.5字节
```

### 量化效果
```
7B模型：
  - FP32：28GB显存
  - FP16：14GB显存
  - INT8：7GB显存
  - INT4：3.5GB显存

速度提升：
  - FP16：比FP32快2倍
  - INT8：比FP16快1.5-2倍
  - INT4：比INT8快1.5-2倍
```

### 量化类型

#### 1. 整数量化（Integer Quantization）
```
方法：将浮点数映射到整数
类型：INT8、INT4
优点：硬件支持好，速度快
缺点：精度损失可能较大
```

#### 2. 浮点数量化（Float Quantization）
```
方法：使用更少的位数表示浮点数
类型：FP16、BF16、FP8
优点：精度损失小
缺点：需要硬件支持
```

#### 3. 混合精度量化
```
方法：不同层使用不同精度
策略：重要层用高精度，次要层用低精度
优点：平衡精度和效率
```

### 量化算法

#### GPTQ
```
类型：后训练量化（PTQ）
特点：基于Hessian矩阵
优点：精度损失小
适用：LLM推理
```

#### AWQ
```
类型：后训练量化
特点：激活感知量化
优点：保留重要权重
适用：4-bit量化
```

#### GGUF/GGML
```
类型：后训练量化
特点：CPU优化，支持多种格式
优点：CPU推理友好
适用：llama.cpp项目
```

#### BitsAndBytes
```
类型：训练和推理量化
特点：支持4-bit和8-bit
优点：Hugging Face集成
适用：QLoRA训练
```

### 量化代码示例

#### Hugging Face BitsAndBytes
```python
from transformers import AutoModelForCausalLM, BitsAndBytesConfig

# 4-bit量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.float16
)

# 加载量化模型
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    quantization_config=bnb_config
)
```

#### GGUF量化
```python
from llama_cpp import Llama

# 加载GGUF量化模型
llm = Llama(
    model_path="llama-2-7b.Q4_K_M.gguf",
    n_ctx=2048,
    n_gpu_layers=35  # GPU层数
)

# 生成文本
output = llm("Hello, how are you?", max_tokens=100)
```

## KV Cache优化

### 什么是KV Cache？
**核心思想**：缓存之前的计算结果，避免重复计算

### 问题背景
```
传统方法：每次生成都重新计算所有token
计算复杂度：O(n²)（自注意力）
生成长度n时：需要计算n²次
```

### KV Cache解决方案
```
方法：缓存Key和Value矩阵
新token：只计算当前token的KV
复用：直接使用缓存的KV
加速：10倍以上
```

### 实现原理
```
步骤1：计算所有token的K和V
步骤2：缓存到内存中
步骤3：生成新token时，只计算新token的KV
步骤4：将新KV追加到缓存中
步骤5：使用所有KV计算注意力
```

### 内存占用
```
问题：KV Cache占用大量内存
计算：层数 × 头数 × 维度 × 序列长度

示例：
  LLaMA-7B：约1GB/1K token
  LLaMA-70B：约10GB/1K token
```

### 优化技术

#### 1. Flash Attention
```
特点：优化注意力计算
优点：减少内存读写，加速计算
效果：2-4倍加速
```

#### 2. Paged Attention
```
特点：像操作系统分页管理内存
优点：减少内存碎片
适用：vLLM系统
```

#### 3. MQA/GQA
```
MQA：Multi-Query Attention
GQA：Grouped-Query Attention
特点：减少KV头数
优点：减少KV Cache大小
```

## 推理框架优化

### 1. vLLM
```
特点：高吞吐量推理引擎
技术：Paged Attention、连续批处理
优点：比Hugging Face快10-20倍
适用：API服务、批量推理
```

### 2. Text Generation Inference（TGI）
```
特点：Hugging Face官方推理服务器
技术：Flash Attention、张量并行
优点：生产就绪，易于部署
适用：在线服务
```

### 3. TensorRT-LLM
```
特点：NVIDIA官方推理优化
技术：TensorRT优化、量化
优点：极致性能
适用：NVIDIA GPU部署
```

### 4. llama.cpp
```
特点：C++实现的LLM推理
技术：GGUF格式、CPU优化
优点：CPU推理高效，内存占用小
适用：边缘设备、个人电脑
```

### 5. ONNX Runtime
```
特点：跨平台推理优化
技术：图优化、量化
优点：硬件无关，部署灵活
适用：多种硬件平台
```

## 批处理优化

### 1. 连续批处理（Continuous Batching）
```
概念：动态添加新请求
优点：提高GPU利用率
对比：静态批处理需要等待
```

### 2. 请求调度
```
策略：优先处理短请求
目标：最小化平均延迟
算法：SJF（最短作业优先）
```

### 3. 并行生成
```
技术：Speculative Decoding
方法：小模型生成草稿，大模型验证
优点：加速2-3倍
```

## 内存优化技术

### 1. 梯度检查点（Gradient Checkpointing）
```
概念：用计算换内存
方法：不保存中间激活，需要时重新计算
优点：减少内存占用50%
缺点：计算时间增加30%
```

### 2. 激活重计算
```
类似：梯度检查点
适用：训练和推理
```

### 3. 内存卸载（Offloading）
```
概念：将不活跃的数据移到CPU/硬盘
优点：处理超大模型
缺点：需要时加载，有延迟
```

## 推理加速技术

### 1. 算子融合（Operator Fusion）
```
概念：合并多个操作为一个
优点：减少内存读写，加速计算
示例：LayerNorm + Residual + Add
```

### 2. 内核优化
```
概念：优化CUDA内核
优点：充分利用硬件特性
示例：Flash Attention、xformers
```

### 3. 稀疏化
```
概念：将不重要的权重设为0
优点：减少计算量
类型：结构化稀疏、非结构化稀疏
```

## 实战优化示例

### 优化LLaMA-7B推理
```python
# 1. 量化
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    load_in_4bit=True
)

# 2. 使用vLLM
from vllm import LLM, SamplingParams

llm = LLM(
    model="meta-llama/Llama-2-7b-hf",
    quantization="awq",
    max_model_len=2048
)

# 3. 批量推理
prompts = ["Hello", "How are you", "Tell me a story"]
sampling_params = SamplingParams(temperature=0.8, top_p=0.95)
outputs = llm.generate(prompts, sampling_params)
```

### 优化GPT-2推理
```python
# 使用TorchScript优化
import torch
from transformers import GPT2LMHeadModel, GPT2Tokenizer

model = GPT2LMHeadModel.from_pretrained("gpt2")
tokenizer = GPT2Tokenizer.from_pretrained("gpt2")

# 转换为TorchScript
example_input = tokenizer("Hello", return_tensors="pt")
traced_model = torch.jit.trace(
    model,
    (example_input["input_ids"], example_input["attention_mask"])
)

# 优化
traced_model = torch.jit.optimize_for_inference(traced_model)
```

## 性能评估指标

### 1. 吞吐量（Throughput）
```
定义：每秒处理的token数
单位：tokens/sec
目标：越高越好
```

### 2. 延迟（Latency）
```
定义：生成第一个token的时间
单位：毫秒
目标：越低越好
```

### 3. 显存占用（Memory Usage）
```
定义：模型和缓存占用的内存
单位：GB
目标：越小越好
```

### 4. 能耗（Energy Consumption）
```
定义：推理消耗的电能
单位：焦耳/token
目标：越低越好
```

## 优化策略选择

### 根据硬件选择
```
高端GPU（A100/H100）：
  - 使用BF16/FP16
  - 启用Flash Attention
  - 批量推理

消费级GPU（RTX 3090/4090）：
  - 使用INT8/INT4量化
  - 限制序列长度
  - 使用vLLM/TGI

CPU：
  - 使用GGUF格式
  - llama.cpp
  - 限制模型大小
```

### 根据应用场景选择
```
在线服务：
  - 低延迟优先
  - 使用量化和缓存
  - 负载均衡

批量处理：
  - 高吞吐量优先
  - 使用批处理优化
  - 并行处理

边缘设备：
  - 内存限制严格
  - 使用小模型+量化
  - CPU推理
```
