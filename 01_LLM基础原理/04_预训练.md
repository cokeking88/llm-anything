# 预训练原理

## 什么是预训练？

### 一句话定义
**一句话**：在海量文本上训练模型，让它学会语言规律

### 类比理解
```
预训练 = 大学通识教育（学语言、学知识）
微调 = 职业培训（学专业技能）

通用GPT → 医疗GPT（医疗数据微调）
通用GPT → 代码GPT（代码数据微调）
```

## 预训练的重要性

### 1. 学习语言规律
```
语法：如何组织词语
语义：词语的含义
语用：如何使用语言
```

### 2. 积累世界知识
```
事实：历史事件、科学知识
常识：日常生活知识
推理：逻辑关系
```

### 3. 涌现能力
```
规模效应：参数越大，能力越强
涌现：小模型没有，大模型突然出现的能力
示例：
  - 小模型：简单问答
  - 大模型：复杂推理、代码生成
```

## 预训练任务类型

### 1. 自回归语言建模（Autoregressive LM）

#### 核心思想
```
预测下一个词
从左到右生成
GPT系列使用
```

#### 示例
```
输入：The cat sat on the
目标：mat

模型学习：P("mat" | "The cat sat on the")
```

#### 公式
```
P(w₁, w₂, ..., wₙ) = ∏ P(wₜ | w₁, w₂, ..., wₜ₋₁)
```

### 2. 掩码语言建模（Masked LM）

#### 核心思想
```
预测被掩盖的词
双向理解上下文
BERT系列使用
```

#### 示例
```
输入：The [MASK] sat on the mat
目标：cat

模型学习：P("cat" | "The [MASK] sat on the mat")
```

#### 变体
```
MLM：随机掩盖15%的词
SpanBERT：掩盖连续片段
ELECTRA：判别器判断词是否被替换
```

### 3. 序列到序列建模（Seq2Seq）

#### 核心思想
```
输入序列 → 输出序列
编码器-解码器架构
T5、BART使用
```

#### 示例
```
输入：translate English to German: How are you?
输出：Wie geht es Ihnen?

输入：summarize: [长文本]
输出：[摘要]
```

## 预训练数据

### 数据来源

#### 1. 网页数据
```
Common Crawl：PB级网页数据
特点：规模大，质量参差不齐
处理：过滤、去重、清洗
```

#### 2. 书籍数据
```
Books3：大量书籍文本
特点：高质量，结构完整
优点：长文本，连贯性好
```

#### 3. 学术论文
```
arXiv：学术论文
特点：专业性强，逻辑严谨
适用：科学、技术领域
```

#### 4. 代码数据
```
GitHub：开源代码
特点：结构化，逻辑性强
适用：代码生成任务
```

#### 5. 维基百科
```
Wikipedia：百科全书
特点：高质量，事实性强
适用：知识问答
```

### 数据规模

#### 不同模型的训练数据
```
GPT-3：300B token

Qwen：3T token
DeepSeek：14T token
```

#### 数据增长趋势
```
2018：GB级
2020：TB级
2023：PB级
趋势：数据量持续增长
```

## 预训练流程

### 1. 数据预处理
```
步骤1：数据收集
步骤2：数据清洗（去重、过滤）
步骤3：Tokenization
步骤4：构建训练样本
```

### 2. 模型初始化
```
方法1：随机初始化
方法2：从头训练
方法3：继续训练（continual pretraining）
```

### 3. 训练过程
```
步骤1：前向传播计算损失
步骤2：反向传播计算梯度
步骤3：优化器更新参数
步骤4：重复直到收敛
```

### 4. 训练技巧

#### 混合精度训练
```
FP32：高精度计算
FP16/BF16：低精度计算
优点：节省显存，加速训练
```

#### 梯度累积
```
概念：累积多个batch的梯度再更新
优点：模拟大batch size
```

#### 学习率调度
```
Warmup：学习率逐渐增加
Decay：学习率逐渐减少
Cosine：余弦退火
```

## 预训练挑战

### 1. 计算资源
```
挑战：需要大量GPU/TPU
成本：数百万美元
时间：数周到数月
```

### 2. 数据质量
```
挑战：网络数据噪声大
解决：数据清洗、过滤
平衡：质量 vs 数量
```

### 3. 训练稳定性
```
挑战：大模型训练不稳定
解决：梯度裁剪、学习率调整
监控：损失曲线、梯度范数
```

### 4. 能源消耗
```
挑战：碳排放大
解决：绿色能源、效率优化
关注：可持续发展
```

## 预训练评估

### 1. 困惑度（Perplexity）
```
定义：模型对测试数据的困惑程度
计算：PPL = exp(-1/N × ∑log P(wₜ))
意义：越低越好
```

### 2. 下游任务评估
```
分类：GLUE、SuperGLUE
问答：SQuAD、Natural Questions
推理：HellaSwag、ARC
代码：HumanEval、MBPP
```

### 3. 生成质量
```
流畅性：生成是否自然
连贯性：逻辑是否一致
多样性：是否重复
```

## 预训练 vs 微调

### 预训练
```
目标：学习通用语言表示
数据：海量无标注数据
任务：语言建模
结果：通用基础模型
```

### 微调
```
目标：适应特定任务
数据：少量有标注数据
任务：分类、生成等
结果：任务专用模型
```

### 关系
```
预训练 + 微调 = 完整学习过程
预训练提供基础能力
微调提供专业技能
```
