# 向量数据库

## 什么是向量数据库？

### 一句话定义
**一句话**：存储和检索向量的数据库

### 类比理解
```
传统数据库：按关键词搜索（精确匹配）
向量数据库：按含义搜索（语义匹配）

搜索"苹果"：
传统：只返回包含"苹果"的结果
向量：还返回"水果"、"iPhone"等相关结果
```

## 向量数据库原理

### 核心概念
```
1. 向量：高维空间中的点，表示文本的语义
2. 距离：向量间的相似度（余弦距离、欧氏距离）
3. 索引：快速查找最近邻向量的数据结构
4. 搜索：找到与查询向量最相似的向量
```

### 工作流程
```
1. 文本 → 嵌入模型 → 向量
2. 向量存储到数据库
3. 查询 → 嵌入模型 → 查询向量
4. 数据库搜索相似向量
5. 返回最相似的向量和对应文本
```

## 主流向量数据库

### 1. FAISS（Facebook AI Similarity Search）
```
特点：本地、快速、开源
优点：性能好，支持GPU
缺点：需要手动管理，无托管服务
适用：小规模、实验、研究
```

### 2. Chroma
```
特点：简单易用、轻量级
优点：API简单，适合原型开发
缺点：功能相对简单
适用：原型开发、小规模应用
```

### 3. Pinecone
```
特点：云服务、托管、易扩展
优点：无需管理，自动扩展
缺点：需要付费，数据在云端
适用：生产环境、大规模应用
```

### 4. Milvus
```
特点：开源、可扩展、功能丰富
优点：支持分布式，功能全面
缺点：部署复杂，需要运维
适用：大规模部署、企业应用
```

### 5. Weaviate
```
特点：开源、功能丰富、GraphQL API
优点：支持混合搜索，易于扩展
缺点：相对较新
适用：需要高级搜索功能的应用
```

### 6. Qdrant
```
特点：开源、高性能、Rust编写
优点：速度快，内存效率高
缺点：相对较新
适用：高性能要求的应用
```

## 数据库选择指南

### 选择标准
```
1. 规模：数据量大小
2. 性能：查询速度要求
3. 成本：预算限制
4. 易用性：开发和维护成本
5. 功能：需要的特性
```

### 场景推荐
| 场景 | 推荐数据库 | 原因 |
|------|------------|------|
| 原型开发 | Chroma | 简单易用，快速启动 |
| 小规模生产 | FAISS | 免费，性能好 |
| 中规模生产 | Pinecone | 托管服务，易于扩展 |
| 大规模生产 | Milvus | 可扩展，功能全面 |
| 多语言支持 | Weaviate | 内置多语言支持 |

## 实战示例

### 使用FAISS
```python
from langchain.vectorstores import FAISS
from langchain.embeddings import OpenAIEmbeddings
from langchain.text_splitter import RecursiveCharacterTextSplitter

# 1. 准备文档
documents = load_documents()
text_splitter = RecursiveCharacterTextSplitter(chunk_size=500)
chunks = text_splitter.split_documents(documents)

# 2. 创建嵌入
embeddings = OpenAIEmbeddings()

# 3. 创建FAISS索引
vectorstore = FAISS.from_documents(chunks, embeddings)

# 4. 保存
vectorstore.save_local("faiss_index")

# 5. 加载
vectorstore = FAISS.load_local("faiss_index", embeddings)

# 6. 搜索
results = vectorstore.similarity_search("机器学习", k=3)
```

### 使用Chroma
```python
import chroma

# 1. 创建客户端
chroma_client = chroma.Client()

# 2. 创建集合
collection = chroma_client.create_collection("documents")

# 3. 添加文档
collection.add(
    documents=["文档1内容", "文档2内容", "文档3内容"],
    ids=["1", "2", "3"],
    metadatas=[{"source": "doc1"}, {"source": "doc2"}, {"source": "doc3"}]
)

# 4. 查询
results = collection.query(
    query_texts=["查询问题"],
    n_results=3
)

print(results)
```

### 使用Pinecone
```python
from pinecone import Pinecone, ServerlessSpec
from langchain.vectorstores import Pinecone as LC_Pinecone

# 1. 初始化
pc = Pinecone(api_key="your-api-key")

# 2. 创建索引
if "documents" not in pc.list_indexes().names():
    pc.create_index(
        name="documents",
        dimension=1536,
        metric="cosine",
        spec=ServerlessSpec(cloud="aws", region="us-west-2")
    )

# 3. 连接索引
index = pc.Index("documents")

# 4. 创建向量存储
vectorstore = LC_Pinecone.from_documents(
    chunks,
    embeddings,
    index_name="documents"
)

# 5. 搜索
results = vectorstore.similarity_search("查询", k=3)
```

## 向量搜索技术

### 搜索类型

#### 1. 相似度搜索
```python
# 基于向量相似度
results = vectorstore.similarity_search(
    query="机器学习",
    k=3,
    filter={"source": "tutorial"}  # 过滤条件
)
```

#### 2. 最大边际相关性（MMR）
```python
# 平衡相关性和多样性
results = vectorstore.max_marginal_relevance_search(
    query="机器学习",
    k=3,
    fetch_k=10  # 先获取10个，再选3个最多样化的
)
```

#### 3. 混合搜索
```python
from langchain.retrievers import EnsembleRetriever
from langchain.retrievers import BM25Retriever

# 向量检索
vector_retriever = vectorstore.as_retriever()

# 关键词检索
bm25_retriever = BM25Retriever.from_documents(chunks)

# 混合检索
ensemble_retriever = EnsembleRetriever(
    retrievers=[bm25_retriever, vector_retriever],
    weights=[0.4, 0.6]
)
```

#### 1. 索引类型
```python
# FAISS索引类型
```

## 监控和维护

### 1. 性能监控
```python
def monitor_vectorstore(vectorstore):
    """监控向量数据库性能"""
    
    # 查询延迟
    start_time = time.time()
    results = vectorstore.similarity_search("测试", k=3)
    latency = time.time() - start_time
    
    # 索引统计
    stats = {
        "total_vectors": vectorstore.index.ntotal,
        "dimension": vectorstore.index.d,
        "query_latency": latency
    }
    
    return stats
```

### 2. 数据维护
```python
def maintain_vectorstore(vectorstore):
    """维护向量数据库"""
    
    # 清理过期数据
    vectorstore.delete(
        filter={"date": {"$lt": "2024-01-01"}}
    )
    
    # 重建索引（如果需要）
    if vectorstore.index.ntotal > 1000000:
        vectorstore.rebuild_index()
```
