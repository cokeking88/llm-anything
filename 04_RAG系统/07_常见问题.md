# 常见问题解答

## RAG基础问题

### Q1：什么是RAG？
**RAG**（Retrieval-Augmented Generation）= 检索增强生成

**工作原理**：
1. 检索：从知识库中找到相关文档
2. 生成：基于检索到的文档生成答案

**优势**：
- 减少幻觉（基于真实文档）
- 知识可更新（更新知识库即可）
- 可解释性（可以查看引用来源）

### Q2：RAG vs 微调怎么选？
| 方面 | RAG | 微调 |
|------|-----|------|
| 知识更新 | 实时更新 | 需要重新训练 |
| 幻觉风险 | 低 | 中等 |
| 成本 | 较低 | 较高 |
| 适用场景 | 知识问答 | 风格/能力调整 |

**建议**：知识频繁更新 → RAG；风格调整 → 微调

### Q3：RAG系统需要多少文档？
**建议**：
- 最小：10-20个文档
- 推荐：100-1000个文档
- 理想：1000+个文档

**注意**：文档质量比数量更重要

## 文档处理问题

### Q4：文档分块多大合适？
**推荐**：
- 一般文档：500-1000字符
- 代码文档：300-500字符
- 书籍章节：1000-2000字符

**原则**：保持语义完整性，避免信息割裂

### Q5：需要多少重叠？
**建议**：50-200字符

**原因**：
- 避免信息割裂
- 保持上下文连续性
- 提高检索准确性

### Q6：如何处理PDF文档？
```python
from langchain.document_loaders import PyPDFLoader

# 方法1：PyPDFLoader
loader = PyPDFLoader("document.pdf")
documents = loader.load()

# 方法2：PyMuPDFLoader（更好的格式保持）
from langchain.document_loaders import PyMuPDFLoader
loader = PyMuPDFLoader("document.pdf")
```

## 向量数据库问题

### Q7：选哪个向量数据库？
**推荐**：
- 开发/测试：Chroma（简单）
- 生产环境小规模：FAISS（免费）
- 生产环境大规模：Pinecone（托管）或Milvus（自建）

### Q8：嵌入模型怎么选？
**推荐**：
- 中文：BGE-large-zh
- 英文：text-embedding-ada-002
- 多语言：E5-large-v2
- 轻量：all-MiniLM-L6-v2

### Q9：向量搜索速度慢怎么办？
**优化**：
1. 使用索引（IVF、HNSW）
2. 减少向量维度
3. 使用GPU加速
4. 实现缓存

## 检索问题

### Q10：检索结果不相关怎么办？
**解决方法**：
1. 调整分块策略
2. 使用混合检索
3. 添加重排序
4. 优化嵌入模型
5. 改进查询理解

### Q11：如何提高召回率？
**方法**：
1. 增加检索数量（k值）
2. 使用MMR搜索
3. 查询扩展
4. 混合检索

### Q12：需要重排序吗？
**建议**：
- 简单场景：不需要
- 生产环境：推荐使用
- 精度要求高：必须使用

## 生成问题

### Q13：模型产生幻觉怎么办？
**解决方法**：
1. 要求基于上下文回答
2. 添加引用溯源
3. 答案验证
4. 降低温度参数

### Q14：回答太长/太短怎么控制？
**方法**：
1. 控制max_tokens
2. 在提示中指定长度
3. 添加长度要求

### Q15：如何实现多轮对话？
```python
from langchain.memory import ConversationBufferMemory
from langchain.chains import ConversationalRetrievalChain

memory = ConversationBufferMemory(memory_key="chat_history")
chain = ConversationalRetrievalChain.from_llm(
    llm=llm,
    retriever=retriever,
    memory=memory
)
```

## 性能问题

### Q16：RAG系统响应慢怎么办？
**优化**：
1. 向量搜索优化
2. 缓存热点查询
3. 异步处理
4. 使用更快的嵌入模型
5. 减少上下文长度

### Q17：如何监控RAG系统？
**监控指标**：
1. 查询延迟
2. 检索准确率
3. 生成质量
4. 用户满意度
5. 错误率

### Q18：需要多少资源？
**建议配置**：
- 开发环境：4核CPU，8GB内存
- 生产环境小规模：8核CPU，16GB内存
- 生产环境大规模：多节点，GPU加速
