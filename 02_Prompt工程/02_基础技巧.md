# 基础技巧

## 零样本学习（Zero-Shot）

### 什么是零样本学习？
**定义**：不给任何例子，直接让模型完成任务

### 示例
```
请将以下英文翻译成中文：
"Hello, how are you?"
```

### 适用场景
```
1. 简单任务
2. 模型擅长的任务
3. 通用任务
4. 时间紧迫时
```

### 优点
```
1. 简单直接
2. 不需要准备例子
3. 适合通用任务
```

### 缺点
```
1. 复杂任务效果差
2. 特定格式难以控制
3. 领域特定任务不理想
```

## 少样本学习（Few-Shot）

### 什么是少样本学习？
**定义**：给几个例子，让模型模仿

### 示例
```
请将以下英文翻译成中文：

示例1：
EN: Good morning → CN: 早上好

示例2：
EN: Thank you → CN: 谢谢

现在翻译：
EN: How are you? → CN: 
```

### 适用场景
```
1. 复杂任务
2. 需要特定格式
3. 领域特定任务
4. 模型表现不佳时
```

### 设计原则
```
1. 例子质量：高质量例子
2. 例子数量：3-5个通常足够
3. 多样性：覆盖不同情况
4. 一致性：格式统一
```

### 优点
```
1. 提高准确性
2. 控制输出格式
3. 适应特定领域
4. 减少幻觉
```

### 缺点
```
1. 需要准备例子
2. 增加token消耗
3. 可能过拟合例子
```

## 角色扮演

### 什么是角色扮演？
**定义**：让模型扮演特定角色，获得专业回答

### 示例
```
你是一位经验丰富的医生。
请用通俗易懂的语言解释高血压的成因。
```

### 角色设计要素
```
1. 专业背景：医生、工程师、作家
2. 经验水平：专家、初学者、老师
3. 性格特点：友好、严谨、幽默
4. 知识领域：特定领域专家
```

### 常见角色
```
1. 技术专家：程序员、架构师、数据科学家
2. 领域专家：医生、律师、金融顾问
3. 创意角色：作家、设计师、策划
4. 教育角色：老师、导师、教练
```

### 优点
```
1. 获得专业回答
2. 控制回答风格
3. 提高相关性
4. 增加可信度
```

### 注意事项
```
1. 角色要合理
2. 避免角色冲突
3. 明确角色边界
4. 注意伦理限制
```

## 提供上下文

### 什么是上下文？
**定义**：提供任务背景和相关信息

### 上下文类型
```
1. 用户背景：用户的知识水平
2. 任务背景：任务的目的和环境
3. 数据背景：数据的来源和含义
4. 领域背景：特定领域的知识
```

### 示例
```
❌ "解释这个代码"
✅ "我是一个Python初学者，请用简单的语言解释这段代码的功能"

上下文包含：
  - 用户背景：Python初学者
  - 任务目标：理解代码功能
  - 语言要求：简单易懂
```

### 上下文设计
```
1. 相关性：只提供相关信息
2. 完整性：提供足够信息
3. 清晰性：表达清晰
4. 简洁性：避免冗余
```

## 指定格式

### 为什么指定格式？
```
原因：
  1. 便于解析和处理
  2. 确保一致性
  3. 满足特定需求
  4. 提高可读性
```

### 常见格式
```
1. 文本格式：段落、列表、标题
2. 结构化格式：JSON、XML、YAML
3. 表格格式：表格、矩阵
4. 代码格式：代码块、语法高亮
```

### 示例
```
❌ "列出优点"
✅ "用5个要点列出优点，每个要点不超过20字，使用Markdown列表"

格式要求：
  - 类型：Markdown列表
  - 数量：5个要点
  - 长度：每个不超过20字
```

### 格式控制技巧
```
1. 明确指定：直接说明格式要求
2. 示例引导：给出格式示例
3. 模板填充：提供模板
4. 约束条件：长度、结构限制
```

## 温度控制

### 什么是温度？
**定义**：控制模型生成随机性的参数

### 温度范围
```
0.0-0.3：确定性强，重复性高
0.4-0.7：平衡创造性和准确性
0.8-1.2：创造性强，多样性高
1.3-2.0：非常随机，可能不连贯
```

### 使用场景
```
低温（0.1-0.3）：
  - 事实性问答
  - 代码生成
  - 数据分析

中温（0.4-0.7）：
  - 通用任务
  - 文本生成
  - 对话

高温（0.8-1.2）：
  - 创意写作
  - 头脑风暴
  - 故事创作
```

### 示例代码
```python
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4",
    temperature=0.2,  # 低温，确定性强
    messages=[
        {"role": "user", "content": "解释量子计算"}
    ]
)
```

## 最大长度控制

### 什么是最大长度？
**定义**：限制生成文本的最大token数

### 设置原则
```
1. 根据任务需求：摘要短，报告长
2. 考虑成本：token越多成本越高
3. 平衡质量：太短可能不完整，太长可能冗余
```

### 常见设置
```
短任务：100-300 tokens
中等任务：300-1000 tokens
长任务：1000-2000 tokens
详细任务：2000-4000 tokens
```

### 示例
```python
response = client.chat.completions.create(
    model="gpt-4",
    max_tokens=500,  # 最大500个token
    messages=[
        {"role": "user", "content": "写一个短故事"}
    ]
)
```

## Stop序列

### 什么是Stop序列？
**定义**：指定停止生成的特定序列

### 使用场景
```
1. 结构化输出：遇到特定标记停止
2. 多段生成：每个段落独立生成
3. 控制长度：遇到结束标记停止
```

### 示例
```python
response = client.chat.completions.create(
    model="gpt-4",
    stop=["\n\n", "END"],  # 遇到双换行或END停止
    messages=[
        {"role": "user", "content": "列出3个优点"}
    ]
)
```
