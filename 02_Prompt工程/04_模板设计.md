# 模板设计

## Prompt模板结构

### 基本结构
```
# 角色
你是一个{角色描述}

# 任务
请{具体任务}

# 要求
1. {要求1}
2. {要求2}
3. {要求3}

# 格式
{输出格式要求}
```

### 完整模板要素
```
1. 角色定义：模型应该扮演什么角色
2. 任务描述：要完成什么任务
3. 输入数据：提供什么数据或信息
4. 输出要求：期望什么格式和风格
5. 限制条件：什么不能做或注意事项
6. 示例说明：可选，提供示例
```

## 万能模板

### 通用模板
```
你是一个{角色}。

任务：{详细任务描述}

输入：
{输入数据或信息}

要求：
1. {要求1}
2. {要求2}
3. {要求3}

输出格式：
{具体格式要求，如JSON、列表等}

注意事项：
{任何限制或特别说明}
```

### 模板示例：文本分析
```
你是一个文本分析专家。

任务：分析以下文章的情感倾向和主要观点

输入：
文章：{文章内容}

要求：
1. 分析情感倾向（积极/消极/中性）
2. 提取3-5个主要观点
3. 评估文章的可信度
4. 用通俗语言解释

输出格式：
{
  "sentiment": "positive/negative/neutral",
  "key_points": ["观点1", "观点2", "观点3"],
  "credibility": "high/medium/low",
  "explanation": "简要解释"
}

注意事项：
- 保持客观中立
- 基于文章内容分析，不要添加个人观点
```

## 领域专用模板

### 技术文档模板
```
你是一个技术文档工程师。

任务：为以下代码生成技术文档

代码：
```{语言}
{代码内容}
```

要求：
1. 功能描述：这段代码做什么
2. 参数说明：每个参数的含义
3. 返回值：返回什么结果
4. 使用示例：如何调用
5. 注意事项：使用时需要注意什么

输出格式：Markdown格式，包括标题、代码块

文档结构：
## 功能描述
## 参数说明
## 返回值
## 使用示例
## 注意事项
```

### 代码审查模板
```
你是一个高级软件工程师。

任务：审查以下代码，提供改进建议

代码：
```{语言}
{代码内容}
```

审查维度：
1. 代码质量：可读性、可维护性
2. 性能：算法效率、资源使用
3. 安全性：潜在漏洞、安全风险
4. 最佳实践：是否符合行业标准
5. 错误处理：异常情况处理

输出格式：结构化审查报告
{
  "overall_score": "1-10",
  "strengths": ["优点1", "优点2"],
  "issues": [
    {
      "severity": "high/medium/low",
      "category": "category",
      "description": "问题描述",
      "suggestion": "改进建议"
    }
  ],
  "summary": "总体评价"
}
```

## 模板设计原则

### 1. 明确性
```
好的模板：
  "分析以下文本的情感倾向，分为积极、消极、中性三类"
  
差的模板：
  "分析文本"
```

### 2. 结构性
```
使用标题、列表、编号
分段清晰
逻辑顺序
```

### 3. 可配置性
```
使用占位符：{角色}、{任务}、{数据}
模板参数化
便于复用
```

### 4. 完整性
```
包含所有必要信息
避免歧义
考虑边缘情况
```

### 5. 简洁性
```
避免冗余
重点突出
易于理解
```

## 模板优化技巧

### 1. 分层设计
```
基础模板：通用框架
领域模板：特定领域
任务模板：具体任务
```

### 2. 版本控制
```
记录模板版本
A/B测试不同版本
持续优化
```

### 3. 反馈循环
```
收集使用反馈
分析失败案例
改进模板
```

### 4. 模板库
```
建立模板库
分类管理
方便检索和复用
```

## 实战模板示例

### 会议纪要模板
```
你是一个专业的会议记录员。

任务：根据以下会议记录，生成结构化的会议纪要

会议记录：
{原始会议记录}

要求：
1. 提取关键决策
2. 记录行动项（谁、什么、何时）
3. 总结主要讨论点
4. 标注未决问题
5. 明确下一步计划

输出格式：
## 会议信息
- 日期：
- 参与者：
- 主题：

## 关键决策
1. 决策1
2. 决策2

## 行动项
| 行动项 | 负责人 | 截止日期 | 状态 |
|--------|--------|----------|------|

## 主要讨论点
1. 讨论点1
2. 讨论点2

## 未决问题
1. 问题1
2. 问题2

## 下一步计划
1. 计划1
2. 计划2
```

## 模板测试和验证

### 测试方法
```
1. 单元测试：测试模板各个部分
2. 集成测试：测试完整模板
3. 边界测试：测试边界情况
4. A/B测试：比较不同模板效果
```

### 验证指标
```
1. 准确性：输出是否符合要求
2. 完整性：是否包含所有信息
3. 一致性：多次运行结果一致
4. 效率：生成速度和资源消耗
```
