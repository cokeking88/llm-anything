# Prompt工程基础

## 什么是Prompt工程？

### 一句话定义
**一句话**：Prompt工程就是"如何问问题"，让AI给出更好的答案

### 类比理解
```
差的提问："帮我写点东西"
好的提问："帮我写一封正式的求职邮件，应聘产品经理职位，强调3年经验"

同样的AI，不同的提问方式，结果天差地别
```

## Prompt工程的重要性

### 1. 解锁模型能力
```
同样的模型，好的Prompt可以：
  - 提高准确性
  - 减少幻觉
  - 获得更相关的回答
  - 控制输出格式
```

### 2. 成本效益
```
优化Prompt vs 重新训练模型：
  - Prompt优化：快速、低成本
  - 模型训练：慢、高成本
  - 结果：Prompt优化通常足够
```

### 3. 广泛适用
```
适用于：
  - 所有LLM（GPT、Claude、LLaMA等）
  - 所有任务（生成、分类、推理等）
  - 所有领域（编程、写作、分析等）
```

## 基本原则

### 1. 清晰具体
```
❌ "写一篇关于AI的文章"
✅ "写一篇800字的文章，介绍AI在医疗领域的三个应用，每个应用200字"

具体要求：
  - 明确任务：做什么
  - 具体要求：怎么做
  - 限制条件：不做什幺
```

### 2. 提供上下文
```
❌ "解释这个代码"
✅ "我是一个Python初学者，请用简单的语言解释这段代码的功能"

上下文要素：
  - 用户背景：谁在问
  - 目的背景：为什么问
  - 领域背景：什么领域
```

### 3. 指定格式
```
❌ "列出优点"
✅ "用5个要点列出优点，每个要点不超过20字，使用Markdown列表"

格式要求：
  - 输出格式：JSON、列表、段落
  - 结构要求：标题、子标题
  - 风格要求：正式、 informal、技术性
```

### 4. 分步指导
```
❌ "解决这个问题"
✅ "请按照以下步骤解决：
    1. 分析问题
    2. 提出假设
    3. 验证假设
    4. 得出结论"

分步好处：
  - 逻辑清晰
  - 易于理解
  - 减少错误
```

## Prompt设计要素

### 1. 任务描述
```
内容：要做什么任务
要求：清晰、具体、无歧义
示例：
  "请总结以下文章的主要内容"
  "请分析这个数据的趋势"
  "请生成一个Python函数"
```

### 2. 输入数据
```
内容：提供给模型的数据
要求：相关、完整、格式清晰
示例：
  "文章：{文章内容}"
  "数据：{数据}"
  "代码：{代码}"
```

### 3. 输出要求
```
内容：期望的输出格式和风格
要求：具体、可衡量
示例：
  "用3个要点总结"
  "输出JSON格式"
  "使用中文回答"
```

### 4. 限制条件
```
内容：限制和约束
要求：明确、必要
示例：
  "不超过200字"
  "不要使用专业术语"
  "只输出结果，不要解释"
```

## Prompt类型

### 1. 指令型Prompt
```
特点：直接给出指令
示例：
  "请翻译以下英文为中文"
  "请总结这段文字"
  "请回答这个问题"
```

### 2. 问答型Prompt
```
特点：提出问题
示例：
  "什么是Transformer架构？"
  "如何提高模型性能？"
  "这个函数的作用是什么？"
```

### 3. 对话型Prompt
```
特点：模拟对话场景
示例：
  "你是一个技术专家，请解释..."
  "作为产品经理，你会如何..."
  "假如你是医生，建议是..."
```

### 4. 创作型Prompt
```
特点：要求创作内容
示例：
  "写一个科幻故事"
  "创作一首诗歌"
  "设计一个产品方案"
```

## Prompt优化策略

### 1. 迭代优化
```
步骤1：设计初始Prompt
步骤2：测试并评估结果
步骤3：分析不足之处
步骤4：调整Prompt
步骤5：重复直到满意
```

### 2. 对比测试
```
方法：同一任务使用不同Prompt
目的：找到最优Prompt
指标：准确性、相关性、完整性
```

### 3. 错误分析
```
分析：为什么Prompt失败
原因：
  - 指令不清晰
  - 上下文不足
  - 格式不匹配
  - 约束不合理
```

### 4. 模板化
```
目的：重复使用优化后的Prompt
方法：创建可配置的模板
好处：一致性、效率
```

## 常见错误

### 1. 指令模糊
```
❌ "写点东西"
✅ "写一篇800字的技术博客，介绍机器学习基础"
```

### 2. 缺乏上下文
```
❌ "解释这个"
✅ "我是一个初学者，请用简单语言解释这段Python代码"
```

### 3. 格式不明确
```
❌ "列出要点"
✅ "用数字列表列出5个要点，每个要点一句话"
```

### 4. 约束矛盾
```
❌ "详细解释，但不超过100字"
✅ "用3句话简要解释核心概念"
```

## 实践练习

### 练习1：改进Prompt
```
原始Prompt： "写个报告"
改进后： "写一份季度销售报告，包括：1.销售总额 2.同比增长 3.主要产品销量 4.地区分布，使用表格格式"
```

### 练习2：设计Prompt
```
任务：让AI解释技术概念
要求：适合初学者，有例子
Prompt： "你是一个友好的技术导师，请用生活中的例子解释{概念}，假设听众是完全没有技术背景的人"
```
