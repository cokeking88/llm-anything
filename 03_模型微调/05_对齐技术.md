# 对齐技术

## RLHF原理

### 什么是RLHF？
**全称**：Reinforcement Learning from Human Feedback（人类反馈强化学习）

**核心思想**：让人类的偏好指导模型学习

### RLHF流程
```
步骤1：指令微调（SFT）
  模型学习遵循指令
  ↓
步骤2：训练奖励模型（RM）
  人类标注：回答A > 回答B
  ↓
步骤3：强化学习（PPO）
  用奖励模型指导生成
```

### RLHF代码示例
```python
from trl import PPOTrainer, PPOConfig
from transformers import AutoTokenizer, AutoModelForCausalLM

# 配置
config = PPOConfig(
    model_name="gpt2",
    learning_rate=1.41e-5,
)

# 加载模型和分词器
model = AutoModelForCausalLM.from_pretrained(config.model_name)
tokenizer = AutoTokenizer.from_pretrained(config.model_name)

# 创建PPO训练器
ppo_trainer = PPOTrainer(
    config=config,
    model=model,
    tokenizer=tokenizer
)

# 训练循环
for batch in dataloader:
    query_tensors = tokenizer(batch["query"], return_tensors="pt")
    
    # 生成回答
    response = ppo_trainer.generate(query_tensors)
    
    # 计算奖励
    rewards = reward_model(response)
    
    # PPO更新
    stats = ppo_trainer.step(query_tensors, response, rewards)
```

## DPO优化

### 什么是DPO？
**全称**：Direct Preference Optimization（直接偏好优化）

**核心思想**：跳过奖励模型，直接学习偏好

### DPO vs RLHF
```
RLHF：偏好数据 → 奖励模型 → 强化学习
DPO：偏好数据 → 直接优化

DPO更简单，效果相当
```

### DPO代码示例
```python
from trl import DPOTrainer, DPOConfig

# 偏好数据格式
# {"prompt": "问题", "chosen": "好的回答", "rejected": "差的回答"}

# DPO配置
dpo_config = DPOConfig(
    output_dir="./dpo_results",
    learning_rate=5e-5,
    num_train_epochs=3,
    per_device_train_batch_size=4
)

# 创建DPO训练器
trainer = DPOTrainer(
    model=model,
    ref_model=ref_model,
    args=dpo_config,
    train_dataset=dataset,
    tokenizer=tokenizer
)

# 训练
trainer.train()
```

## 对齐方法对比

### 方法对比表
| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| RLHF | 效果好，成熟 | 实现复杂，训练不稳定 | 资源充足，追求最佳效果 |
| DPO | 简单，稳定 | 效果略低于RLHF | 资源有限，快速实现 |
| KTO | 无需偏好对 | 需要大量数据 | 数据充足的场景 |
| IPO | 理论更优 | 实现复杂 | 研究和实验 |

### 选择建议
```
资源充足，追求最佳效果 → RLHF
资源有限，快速实现 → DPO
数据充足，无需偏好对 → KTO
研究实验，理论优化 → IPO
```

## 宪法AI（Constitutional AI）

### 什么是宪法AI？
**核心思想**：让AI自我对齐，遵循一组原则

### 实现方式
```
1. 定义宪法原则（一组规则）
2. AI自我批评和修正
3. 基于修正后的数据训练
```

### 代码示例
```python
# 定义宪法原则
constitution = [
    "回答应该有帮助且无害",
    "回答应该尊重用户隐私",
    "回答应该诚实且不误导",
    "回答应该避免歧视和偏见"
]

# 宪法AI训练
def constitutional_training(model, dataset, constitution):
    for batch in dataset:
        # 生成初始回答
        response = model.generate(batch["prompt"])
        
        # 根据宪法自我批评
        critique = critique_model.generate(
            f"根据以下原则批评这个回答：\n"
            f"原则：{constitution}\n"
            f"回答：{response}"
        )
        
        # 修正回答
        revised_response = revision_model.generate(
            f"根据批评修正回答：\n"
            f"原回答：{response}\n"
            f"批评：{critique}"
        )
```

## 对齐评估

### 评估指标
```python
def evaluate_alignment(model, test_dataset):
    """评估模型对齐程度"""
    
    metrics = {
        "helpfulness": 0,  # 有帮助程度
        "harmlessness": 0,  # 无害程度
        "honesty": 0,       # 诚实程度
        "consistency": 0    # 一致性
    }
    
    for test_case in test_dataset:
        response = model.generate(test_case["prompt"])
        
        # 评估有帮助性
        metrics["helpfulness"] += evaluate_helpfulness(
            response, test_case["expected"]
        )
        
        # 评估无害性
        metrics["harmlessness"] += evaluate_harmlessness(response)
        
        # 评估诚实性
        metrics["honesty"] += evaluate_honesty(
            response, test_case["facts"]
        )
    
    # 计算平均值
    for key in metrics:
        metrics[key] /= len(test_dataset)
    
    return metrics
```

## 对齐挑战

### 1. 对齐税
```
问题：对齐可能降低模型性能
解决：平衡对齐和性能，选择合适的对齐方法
```

### 2. 奖励黑客
```
问题：模型学会欺骗奖励模型
解决：使用更稳健的奖励模型，多维度评估
```

### 3. 可扩展监督
```
问题：随着模型变强，监督变得更难
解决：使用AI辅助监督，分层对齐
```
