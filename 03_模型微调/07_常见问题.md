# 常见问题解答

## 微调基础问题

### Q1：什么是微调？
**微调**（Fine-tuning）是在预训练模型基础上，用特定数据进一步训练，让它更擅长特定任务。

### Q2：微调 vs 预训练有什么区别？
| 方面 | 预训练 | 微调 |
|------|--------|------|
| 数据量 | 海量无标注数据 | 少量有标注数据 |
| 时间 | 数周到数月 | 数小时到数天 |
| 资源 | 大量GPU | 少量GPU |
| 目的 | 学习通用语言能力 | 适应特定任务 |

### Q3：需要多少数据进行微调？
**建议**：
- 简单任务：100-1000条
- 中等任务：1000-10000条
- 复杂任务：10000+条

**原则**：质量比数量更重要

## LoRA相关问题

### Q4：LoRA的秩(r)应该设多少？
**建议**：
- 简单任务：8-16
- 中等任务：16-32
- 复杂任务：32-64

**权衡**：秩越高效果越好，但参数越多

### Q5：LoRA应该应用到哪些层？
**推荐**：
- 基础：q_proj, v_proj
- 全面：q_proj, k_proj, v_proj, o_proj
- 完整：所有线性层

### Q6：如何判断LoRA是否有效？
**检查点**：
1. 训练损失是否下降
2. 验证集性能是否提升
3. 生成样本质量是否提高

## QLoRA相关问题

### Q7：QLoRA需要多少显存？
**参考**：
- 7B模型：约5-6GB
- 13B模型：约8-10GB
- 70B模型：约30-40GB

### Q8：量化会损失多少精度？
**影响**：
- INT4量化：损失约2-5%性能
- INT8量化：损失约1-3%性能

**建议**：QLoRA通常能保持90-95%的性能

## 训练问题

### Q9：训练时loss不下降怎么办？
**可能原因和解决方法**：

1. **学习率太高**：降低学习率到1e-5
2. **数据质量差**：清洗数据，去除低质量样本
3. **批次大小太小**：增加批次大小或使用梯度累积
4. **模型太小**：使用更大的基础模型

### Q10：如何防止过拟合？
**方法**：
1. **早停**：监控验证集性能
2. **Dropout**：设置lora_dropout=0.1-0.2
3. **数据增强**：增加数据多样性
4. **正则化**：使用权重衰减

### Q11：训练需要多长时间？
**参考**（单卡A100）：
- 7B模型，1000条数据：约30分钟
- 13B模型，5000条数据：约2小时
- 70B模型，10000条数据：约8小时

## 对齐技术问题

### Q12：RLHF和DPO哪个更好？
**选择建议**：
- 资源充足，追求最佳效果 → RLHF
- 资源有限，快速实现 → DPO
- 数据充足，无需偏好对 → KTO

### Q13：什么是对齐税？
**定义**：对齐可能降低模型在某些任务上的原始性能

**平衡方法**：
1. 选择合适的对齐强度
2. 多维度评估
3. 渐进式对齐

## 数据相关问题

### Q14：数据格式有什么要求？
**推荐格式**：
```json
{
  "instruction": "任务描述",
  "input": "输入内容（可选）",
  "output": "期望输出"
}
```

### Q15：如何处理数据不平衡？
**方法**：
1. **重采样**：平衡各类别数据量
2. **加权损失**：为少数类别分配更高权重
3. **数据增强**：增加少数类别数据

### Q16：需要验证集吗？
**需要**，用于：
1. 监控训练过程
2. 防止过拟合
3. 选择最佳模型

**比例**：训练集80%，验证集10%，测试集10%

## 模型评估问题

### Q17：如何评估微调效果？
**评估维度**：
1. **任务性能**：目标任务准确率
2. **通用能力**：保持的预训练能力
3. **生成质量**：流畅性、相关性
4. **安全性**：无害性、诚实性

### Q18：如何选择最佳checkpoint？
**建议**：
1. 监控验证集损失
2. 选择验证集性能最好的checkpoint
3. 考虑过拟合情况

## 部署问题

### Q19：如何部署微调后的模型？
**步骤**：
1. 合并LoRA权重（可选）
2. 量化模型（可选）
3. 选择推理框架（vLLM、TGI等）
4. 部署服务

### Q20：部署需要多少资源？
**参考**：
- 7B模型：8GB显存
- 13B模型：16GB显存
- 70B模型：80GB显存

**优化**：使用量化可以减少50-75%的显存需求

## 故障排除

### Q21：CUDA内存不足怎么办？
**解决方法**：
1. 减小batch_size
2. 增加gradient_accumulation_steps
3. 使用更小的模型
4. 使用QLoRA（4-bit量化）
5. 使用梯度检查点

### Q22：训练速度太慢怎么办？
**优化方法**：
1. 使用混合精度（fp16/bf16）
2. 使用Flash Attention
3. 增加gradient_accumulation_steps
4. 使用更高效的硬件
