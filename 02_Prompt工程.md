# 02_Prompt工程

## 什么是Prompt工程？

**一句话**：Prompt工程就是"如何问问题"，让AI给出更好的答案

**类比**：
```
差的提问："帮我写点东西"
好的提问："帮我写一封正式的求职邮件，应聘产品经理职位，强调3年经验"

同样的AI，不同的提问方式，结果天差地别
```

---

## 基本原则

### 1. 清晰具体

```
❌ "写一篇关于AI的文章"
✅ "写一篇800字的文章，介绍AI在医疗领域的三个应用"
```

### 2. 提供上下文

```
❌ "解释这个代码"
✅ "我是一个Python初学者，请用简单的语言解释这段代码"
```

### 3. 指定格式

```
❌ "列出优点"
✅ "用5个要点列出优点，每个要点不超过20字"
```

## Prompt模板

### 万能模板

```
# 角色
你是一个{角色描述}

# 任务
请{具体任务}

# 要求
1. {要求1}
2. {要求2}
3. {要求3}

# 格式
{输出格式要求}
```

### 示例

```
# 角色
你是一个资深的数据分析师

# 任务
请分析这份销售数据，找出销售下滑的原因

# 要求
1. 使用通俗易懂的语言
2. 给出至少3个可能的原因
3. 每个原因都要有数据支持

# 格式
用Markdown列表格式输出
```

## 零样本 vs 少样本

### 零样本（Zero-Shot）

**定义**：不给任何例子，直接让模型完成任务

```
请将以下英文翻译成中文：
"Hello, how are you?"
```

**适用**：简单任务、模型擅长的任务

### 少样本（Few-Shot）

**定义**：给几个例子，让模型模仿

```
请将以下英文翻译成中文：

示例1：
EN: Good morning → CN: 早上好

示例2：
EN: Thank you → CN: 谢谢

现在翻译：
EN: How are you? → CN: 
```

**适用**：复杂任务、需要特定格式

## 思维链（Chain of Thought）

### 什么是思维链？

**一句话**：让模型"一步步思考"，而不是直接给答案

**类比**：
```
普通回答：直接写答案
思维链：写出解题过程

就像数学题，写出过程比只写答案得分更高
```

### 示例

```
❌ 普通Prompt：
问题：15头牛每天产20升奶，一周产多少？
答案：2100升

✅ 思维链Prompt：
问题：15头牛每天产20升奶，一周产多少？
请一步步思考：

答案：
1. 每天产奶 = 15 × 20 = 300升
2. 一周产奶 = 300 × 7 = 2100升
所以一周产2100升。
```

### 技巧

```
添加"让我们一步步思考"
添加"请检查你的答案"
添加推理步骤要求
```

## 高级技巧

### 角色扮演

```
你是一位经验丰富的医生。
请用通俗易懂的语言解释高血压的成因。
```

### 系统提示

```
System: 你是一个友好的AI助手
- 始终用中文回答
- 回答要简洁明了
- 如果不确定，坦诚说明
```

### 负面提示

```
请总结这篇文章：
- 不要超过200字
- 不要使用专业术语
- 不要添加个人观点
```

### 输出格式控制

```
请分析文本情感，以JSON格式返回：
{
  "sentiment": "positive/negative/neutral",
  "confidence": 0.95,
  "reason": "原因"
}
```

## 实战案例

### 案例1：文本总结

```
请总结以下文章：
1. 不超过100字
2. 提取3个关键点
3. 使用通俗语言

文章：{文章内容}
```

### 案例2：代码生成

```
请写一个Python函数：
- 函数名：calculate_fibonacci
- 输入：n（整数）
- 输出：第n个斐波那契数
- 包含类型注解和docstring
```

### 案例3：数据分析

```
你是一个数据分析师，请分析销售数据：
1. 整体趋势
2. 异常值分析
3. 三个关键发现
4. 两条业务建议

数据：{数据}
```

## 常见问题

### Q：Prompt越长越好吗？

**不一定**：
- 简单任务：短prompt即可
- 复杂任务：需要详细prompt
- 注意token限制

### Q：如何提高回答质量？

**技巧**：
1. 给AI"思考时间"（思维链）
2. 提供上下文
3. 指定角色
4. 要求自我验证
5. 使用少样本学习
